## Goal (user POV)
"The whole point is I stop doing it. Set it to fire on a schedule or when something happens, and trust it'll run."

## Definition of done
- I configured a trigger (schedule or event) without guessing, and I can tell it's armed and when it'll next fire.
- I believe it will actually run unattended — there's evidence, not just a saved form.

## What L1 must check
- Trigger config → subscription → run path; can a user tell an enabled trigger from a disabled one?
- Are schedule + event triggers both reachable and legible to a non-dev?

## What L2 must confirm (l2_priority)
- A trigger actually fires and produces a run (or the next-fire time is real).
- Clarity of "is this on?" state at a glance.
