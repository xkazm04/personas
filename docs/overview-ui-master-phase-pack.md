## Executive Summary

- Baseline quality is already strong for list-heavy modules (`executions`, `messages`, `memories`).
- Highest uplift opportunities are `budget`, `realtime`, and `observability` due to consistency/state-contract gaps.
- The recommended path is pass-based: normalize tokens first, then state/accessibility, then module-specific polish.
