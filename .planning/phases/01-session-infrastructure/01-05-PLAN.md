      - Flush (advance RAF timers for Channel, tick for EventBridge)

      - Emit 10 more events (simulates backend continuing while frontend is away)
      - Call reattach (simulates navigation back)
      - Assert: Both handlers lost the 10 events emitted during teardown
      - Assert: Both handlers need SQLite checkpoint hydration to recover state
      - This confirms the CONTEXT.md assessment: "Same checkpoint-based recovery needed"

      - Measure time to process all events

      - EventBridge: `listen<BuildEvent>("build-event", handler)` -- type parameter is a runtime hint, no compile-time enforcement on the emit side
