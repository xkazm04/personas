**In `src-tauri/src/engine/cli_process.rs`:**

1. Add a new method `write_stdin_line` that writes data to stdin WITHOUT consuming the handle. The key difference from `write_stdin`: do NOT call `.take()` on stdin. Instead, borrow it via `self.child.stdin.as_mut()`. Do NOT call `.shutdown()`. Append a newline (`\n`) after the data so the CLI process can detect end-of-input per line.

```rust
/// Write a line to the child's stdin without consuming or closing it.
/// Suitable for multi-turn Q&A where stdin must remain open for subsequent writes.
pub async fn write_stdin_line(&mut self, data: &[u8]) -> Result<(), std::io::Error> {
    if let Some(stdin) = self.child.stdin.as_mut() {
        stdin.write_all(data).await?;
        stdin.write_all(b"\n").await?;
        stdin.flush().await?;
        Ok(())
    } else {
        Err(std::io::Error::new(
            std::io::ErrorKind::BrokenPipe,
            "stdin already consumed or closed",
        ))
    }
}
```

Note: `flush()` is critical -- without it, the data may sit in a buffer and never reach the CLI process.
