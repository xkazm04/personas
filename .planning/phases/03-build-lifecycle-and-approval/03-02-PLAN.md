Wire mandatory test run into the build flow with streaming output, approval/rejection gate, and command center UI.

    **Create src/features/agents/components/matrix/useMatrixLifecycle.ts:**

    This hook orchestrates the post-build lifecycle (test -> approve/reject -> promote). It bridges the existing test infrastructure to the build flow.

    1. Accept `{ personaId: string | null }` parameter
    2. Read build slice state: buildPhase, buildTestId, buildTestPassed, buildTestError, buildTestOutputLines
    3. `handleStartTest()`:
       - Guard: only callable when buildPhase === "draft_ready"
       - Generate testId: `"build-test-" + crypto.randomUUID()`
       - Call `useAgentStore.getState().handleStartTest(testId)` to transition phase
       - Load persona detail via `getPersonaDetail(personaId)` to get tools and model_profile
       - Call `startTestRun(personaId, [{ id: persona.model_profile ?? "default" }])` to begin the test
       - On API error: call `handleTestFailed(errMsg(err, "Failed to start test"))`
    4. `handleApproveTest()`:
       - Guard: only callable when buildPhase === "test_complete" && buildTestPassed === true
       - For now, just return true (Plan 03 will wire the actual promotion)
       - This is the approval signal -- Plan 03's promotion will check this
    5. `handleRejectTest()`:
       - Call `useAgentStore.getState().handleRejectTest()` to return to draft_ready
    6. Use `useRunEventListener` with a binding for "test-run-status":
       - Filter: match testId from buildTestId in the store
       - onProgress: append output to buildTestOutputLines via a new `appendTestOutput(line: string)` action (add to slice)
       - onTerminal: check payload status -- "completed" + passed -> handleTestComplete(true, preview), "failed" -> handleTestFailed(error)
    7. Return: `{ handleStartTest, handleApproveTest, handleRejectTest, buildTestPassed, buildTestError, buildTestOutputLines, isTesting: buildPhase === "testing", isTestComplete: buildPhase === "test_complete" }`

    **NOTE:** Add `appendTestOutput(line: string)` action to matrixBuildSlice -- appends to buildTestOutputLines array (cap at 200 lines). This is a small addition to the slice from Plan 01.

    **Create tests:**
    Tests should mock the API calls (vi.mock for startTestRun, getPersonaDetail) and test the hook's orchestration logic. Since useRunEventListener is a hook, test the binding configuration rather than the full event flow.
