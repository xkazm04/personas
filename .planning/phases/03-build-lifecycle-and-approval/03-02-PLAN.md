<objective>
Wire mandatory test run into the build flow with streaming output, approval/rejection gate, and command center UI including lifecycle orb states.

Purpose: After the matrix is filled (draft_ready), users must be able to test their agent against their use case and see real results before promoting. This plan connects the existing test infrastructure (testN8nDraft API + Tauri listen for n8n-test-status/n8n-test-output events) to the build lifecycle, adds test/approve/reject UI states to the command center (fulfilling LIFE-01 orb rendering for testing/test_complete/promoted phases), and threads lifecycle actions through UnifiedMatrixEntry.

Output: useMatrixLifecycle hook for test/approve/reject orchestration, extended MatrixCommandCenter with 3 new state renderers (TestRunningIndicator, TestResultsPanel, PromotionSuccessIndicator), UnifiedMatrixEntry wired to lifecycle actions.
</objective>

    **Create src/features/agents/components/matrix/useMatrixLifecycle.ts:**

    This hook orchestrates the post-build lifecycle (test -> approve/reject -> promote). It bridges the existing test infrastructure to the build flow. Per RESEARCH.md recommendation, use `testN8nDraft` (single-turn, streaming, confusion detection) rather than `startTestRun` (full multi-model test runner).

    1. Accept `{ personaId: string | null }` parameter
    2. Read build slice state: buildPhase, buildTestId, buildTestPassed, buildTestError, buildTestOutputLines
    3. `handleStartTest()`:
       - Guard: only callable when buildPhase === "draft_ready"
       - Generate testId: `"build-test-" + crypto.randomUUID()`
       - Call `useAgentStore.getState().handleStartTest(testId)` to transition phase to "testing"
       - Load persona detail via `getPersonaDetail(personaId)` to get persona + tools for draft JSON
       - Serialize persona + tools as EphemeralPersona JSON for the draft test
       - Call `validateN8nDraft(draftJson)` first -- if not passed, call handleTestFailed with validation error and return
       - Call `testN8nDraft(testId, draftJson)` to spawn the streaming test (returns immediately, streams via events)
       - On API error: call `handleTestFailed(errMsg(err, "Failed to start test"))`
    4. Set up Tauri event listeners in a useEffect:
       - `listen("n8n-test-status", handler)` -- filter by `event.payload.test_id === buildTestId`
         - status "completed" + passed=true -> `handleTestComplete(true, "")`
         - status "failed" -> `handleTestFailed(event.payload.error ?? "Test failed")`
       - `listen("n8n-test-output", handler)` -- filter by test_id
         - Call `appendTestOutput(event.payload.line)` to stream output to the store
       - Return unlisten cleanup functions from the useEffect
    5. `handleApproveTest()`:
       - Guard: only callable when buildPhase === "test_complete" && buildTestPassed === true
       - For now, just return true -- this is a stub that Plan 03 replaces with handlePromote
       - NOTE: Plan 03 will add `handlePromote()` to this hook which performs credential validation and persona update. The `handleApproveTest` name is intentionally a thin approval signal; Plan 03 renames/replaces it with `handlePromote` which does the real work.
    6. `handleRejectTest()`:
       - Call `useAgentStore.getState().handleRejectTest()` to return to draft_ready
    7. Return: `{ handleStartTest, handleApproveTest, handleRejectTest, buildTestPassed, buildTestError, buildTestOutputLines, isTesting: buildPhase === "testing", isTestComplete: buildPhase === "test_complete" }`

    **NOTE:** Add `appendTestOutput(line: string)` action to matrixBuildSlice -- appends to buildTestOutputLines array (cap at 200 lines). This is a small addition to the slice from Plan 01.

    **Create tests:**
    Tests should mock the API calls (vi.mock for testN8nDraft, validateN8nDraft, getPersonaDetail) and mock Tauri's listen function. Test the hook's orchestration logic including event filtering by test_id.
