<objective>
Implement refinement loop and draft-to-production promotion with credential coverage validation.

Purpose: After testing, users need two paths: (1) refine specific cells by typing feedback that starts a new build session with context, and (2) promote the tested draft to a real production persona. Promotion requires credential coverage validation (INTG-04) and updates the existing draft persona to enabled status with all resolved configuration.

Output: handleRefine and handlePromote actions in useMatrixLifecycle, credential coverage gate check, promotion via updatePersona API, success indicator with navigation.

Naming delegation: Plan 02 created `handleApproveTest()` as a stub returning true. This plan replaces that stub with `handlePromote()` which performs the actual credential validation and persona update. The hook's return changes from `handleApproveTest` to `handlePromote`. The MatrixCommandCenter's `onApproveTest` prop now calls `handlePromote` instead of the stub. This is a clean rename -- the "Approve" button in TestResultsPanel triggers `handlePromote` which does validation + promotion in one step.
</objective>

    **Extend useMatrixLifecycle.ts with two new actions, replacing the handleApproveTest stub:**

    1. `handleRefine(feedback: string)`:
       - Guard: only callable when buildPhase === "draft_ready" (after rejection or initial draft_ready)
       - Read `buildDraft` (agent_ir) from store via `useAgentStore.getState().buildDraft`
       - Construct refinement intent: `"[REFINEMENT] Previous build context: " + JSON.stringify(buildDraft) + "\n\nUser refinement: " + feedback`
       - Call `build.handleGenerate(refinementIntent, personaId)` to start a NEW build session
       - This reuses the existing build session infrastructure -- the CLI's intent_compiler treats the prefixed context as refinement guidance
       - Reset test state since we're going back to building

    2. `handlePromote()` -- **replaces the handleApproveTest stub from Plan 02**:
       - Remove the `handleApproveTest` stub entirely
       - Guard: only callable when buildPhase === "test_complete" && buildTestPassed === true
       - Load persona detail: `getPersonaDetail(personaId)` to get current tools
       - Parse credential_links from buildDraft (agent_ir): extract `credential_links` object if present
       - Run credential coverage check: `computeCredentialCoverage(detail.tools, credentialLinks)`
       - If not covered: return `{ success: false, coverage: result }` -- DO NOT promote
       - If covered: call `updatePersona(personaId, buildUpdateInput({ enabled: true, design_context: JSON.stringify({ credential_links: credentialLinks }), name: agentIR.name, description: agentIR.description }))` using the `buildUpdateInput` helper from personas.ts
       - On success: call `useAgentStore.getState().handleBuildSessionStatus({ type: "session_status", session_id: buildSessionId, phase: "promoted", resolved_count: 8, total_count: 8 })` to transition to promoted
       - Return `{ success: true, coverage: result }`
